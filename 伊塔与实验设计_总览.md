## 0. 最先记住的一句话

> **η（伊塔）不是目的，它只是一把"更精细的尺子"。**
> 我们的目的从头到尾只有一个：**在级联故障中，找出"在哪个时段、做什么动作" 最能救场**。
> η 之所以从原来的"一条公式"拆成"四个因子"，是因为原来那把尺子太粗，看不出动作之间的差别。仅此而已。

这一点一旦动摇，就会陷入"不停改 η"的死循环。**η 的任何改动，都必须能回答一句话：它能不能让关键动作更容易被分辨出来？** 不能就别改。

## 2. η 的角色：让上面四步都看得更清楚

R₁ 本身是"负荷保住比例"。但 R₁ 要想反映"时延影响"，必须通过发电机出力折扣 φ 进来，而 φ 来自 η：

```
           四个因子 η⁺                 功率折扣 φ              最终 R₁
 时延 ─→ Φ_dead·Φ_sat·Φ_loss·Φ_crit ─→ P_actual = η⁺·P_ref ─→ 保住负荷/总负荷
         （每台机一个 η⁺）             （逐机组折扣）
```

**没有 η，时延和 R₁ 就接不起来**——那样画出来的箱线图只是"机器随机涨落"，热力图也是一片噪声。η 的作用是把"秒数"翻译成"出力折扣"，再翻译成 R₁ 的差距。

> 所以 η 不是另起炉灶，它**只是箱线图背后的"计算引擎"**。

## 3. 四个因子 vs 旧时延：一张对照表就够

### 3.1 旧体系的所有时延（全部来自 `createDelayConfig.m`）

| 代号 | 物理含义 | 配置字段 | 量纲 |
|---|---|---|---|
| **D1** 串行时延 | 包长/链路速率 | `packet_size_bits_up/down`, `default_link_rate_bps` | 秒 |
| **D2** 传播时延 | 距离/光速 | `default_distance_km`, `propagation_speed_kmps` | 秒 |
| **D3** CC 服务 | 控制中心节点 tx/rx/forward | `service.cc.*` | 秒 |
| **D4** 非 CC 服务 | 普通节点 tx/rx/forward | `service.noncc.*` | 秒 |
| **D5** 测量时延 | 物理量被采到 | `pb_to_noncc_measurement_delay_s = 0.10` | 秒 |
| **D6** 执行时延 | 控制指令被执行 | `noncc_to_pb_execution_delay_s = 0.12` | 秒 |
| **D7** 测量敏感度 | k_m = 0.80 | `measurement_sensitivity` | 1/秒 |
| **D8** 执行敏感度 | k_e = 0.60 | `execution_sensitivity` | 1/秒 |

旧 η（`computePowerDelayEfficiency.m`）就一行核心：
```
τ_m = (D1+D2+D3+D4 走上行路径累加) + D5
τ_e = (D1+D2+D3+D4 走下行路径累加) + D6
η  = (1 - k_m·τ_m) · (1 - k_e·τ_e)
```
问题：线性乘积，**所有机组共用同一对 k，小延迟、大延迟、长路径、短路径全部被打成一个分数**，动作实验里不同动作的 η 差得很小，关键动作分辨不出来。

### 3.2 新的四个因子 —— 把"一个总账"拆成"四种信号"

| 新因子 | 白话含义 | 接收哪些旧时延 | 为什么要单独拆出来 |
|---|---|---|---|
| **Φ_dead**<br>（死区） | "延迟很小时控制器压根感觉不到" | 用 τ_m、τ_e，减去阈值 τ_m0、τ_e0 | 旧公式里再小的延迟都要扣一点，不符合真实控制系统的死区行为；导致轻延迟场景差距很小 |
| **Φ_sat**<br>（饱和） | "延迟越大越差，但不会瞬间崩到 0" | **D1+D2+D3+D4+D5 → τ_m**；**D1+D2+D3+D4+D6 → τ_e**；**D7→a_m，D8→a_e** | 旧公式 (1-k·τ) 在 τ 大时变负再被截成 0，丢失区分度。用指数形式保持单调、正、有曲率 |
| **Φ_loss**<br>（路径丢包） | "每多一跳就多一次丢包机会" | 只用**路径跳数 H_up + H_dn**（不直接用秒数） | **新增维度**：旧体系完全没有"可靠性"，4 跳和 2 跳的路径只要总时延一样 η 就一样，现实里长路径更不可靠 |
| **Φ_crit**<br>（机组临界） | "大机组抗延迟能力强，小机组先崩" | (τ_m+τ_e) 与机组个性化阈值 τ_crit(i) 比较；τ_crit(i) 由 P_max(i) 给出 | 旧体系所有机共用同一对 k，大机小机一视同仁；现实中大机惯量大、扛得住 |

**最终 η⁺(i) = Φ_sat(i) · Φ_loss(i) · Φ_crit(i)**（Φ_dead 是预处理）

### 3.3 你最关心的那个问题："路径丢包和时延有关吗？"

**答：耦合但不同一。** 用一张图说清：

```
                           一条具体的通信路径 (CC → ... → 机组 i)
                                  │
                ┌─────────────────┴─────────────────┐
                │                                   │
        路径上的时延累加                      路径上的跳数
        (秒：D1+D2+D3+D4 全部秒数相加)       (整数：H)
                │                                   │
                ▼                                   ▼
            进 Φ_sat                           进 Φ_loss
        "慢了多少"                            "不可靠了多少"
```

- **共享的只有"这条路径"**：路径越长，累加的秒数越多，同时跳数 H 也越大。所以"时延大"通常伴随"丢包也多"，但**参数独立**（秒数由 D1..D4 决定；丢包率由 p_hop × H 决定）。
- **不会重复计算**：Φ_sat 只吃秒数；Φ_loss 只吃跳数。两件事物理上是分开的。
- **一句话版本**："同一条路径，两本账。一本记慢了多少秒，一本记漏了多少包。"

### 3.4 "原来设计的时延，哪去了？"一句话总结

> **没有任何旧时延被删。D1-D6 还是按老办法加进 τ_m、τ_e，D7/D8 换了个名字叫 a_m、a_e。四个因子只是重新"分箱"，并把路径跳数 H 和机组容量 P_max 这两个本来就存在但没用上的信息补了进来。**

如果把新参数都退化（τ_m0=τ_e0=0，p_hop=0，τ_crit=∞，β=1），η⁺ 就退化回旧 η 的一阶近似——**向下兼容**。

## 4. 动作实验怎么接下去做（这才是大头）

四个因子设计好以后，我们要干的是下面这件事：

### 4.1 第三步"关键动作"的实验表

| 动作类别 | 具体动作 | 作用于哪个因子 | 预期信号 |
|---|---|---|---|
| **通信侧** | 增加/备份路径（减少 H） | Φ_loss ↑ | 长距离节点 R₁ 拉回 |
| **通信侧** | 提升链路速率（减 D1） | Φ_sat ↑ | 全局小幅改善 |
| **通信侧** | CC 节点优先级调度（减 D3） | Φ_sat ↑ | 关键路径改善 |
| **电力侧** | 提前切负荷 | 减少 P_ref 基数 | R₁ 分母变小，比例升 |
| **电力侧** | 发电机调度（切小机保大机） | Φ_crit（小机先崩，索性不救） | 与 τ_crit 耦合显著 |
| **电力侧** | 增加旋转备用 | 提高 P_max(i) → τ_crit(i) | Φ_crit 整体抬升 |

**关键发现应该长这样**：同一时间窗 (best time from Fig4)，不同动作的 R₁ 恢复量排序稳定 → 告诉我们"在这个时机，最该做 X"。

### 4.2 第四步 2×2 对比实验的预测

用四因子 η⁺ 算出来的 R₁ 差距应该**大于**旧 η 下的差距——这本身就是四因子设计好的证据。具体预期：

|  | 最佳动作 | 最差动作 | 差距(ΔR₁) |
|---|---|---|---|
| **最佳时机** | C1: 最高 | C2: 中等 | 中 |
| **最差时机** | C3: 中等 | C4: 最低 | 中 |
| **时机差距(ΔR₁)** | 中 | 中 | — |

👉 **十字差 (C1-C4)** 如果显著大于任一单维差 (C1-C2 或 C1-C3)，就说明"时机 × 动作"是**非可加的协同效应**——这是论文要打的关键点。

## 5. 如果别人问你……（对话模板）

**问：η 里那四个因子是干嘛的？**
答：把原来"一根棍子打所有延迟"的粗糙公式拆开，让死区、饱和、丢包、机组容量这四件事各走各的账。目的是让不同动作在 R₁ 上的差距变明显、可分辨。

**问：你之前设计的那些时延还用吗？**
答：全部还在用。串行、传播、服务、测量、执行这六种时延还是按老办法累加，只是累加完之后不再进一根棍子，而是进"饱和"这一只因子，同时它们走过的路径跳数另外进"丢包"这一只因子。

**问：路径丢包跟时延什么关系？**
答：它俩**共享同一条路径但算两本账**。时延记秒数，丢包记跳数。路径越长两边都变差，但参数独立。

**问：为什么不直接用 R₁，还要搞 η？**
答：R₁ 只看"最终保住多少负荷"。但时延不会直接切掉负荷，它先让发电机出力打折（出力折扣就是 η），才传导到 R₁。没有 η 这一步翻译，R₁ 对时延就不敏感、分辨不出动作差别。

**问：整个研究最后要证明什么？**
答：证明**在级联故障中，干预的"时机"和"动作"是两个独立且协同的维度**，最佳组合比最差组合能把 R₁ 救回显著更多。η 四因子是让这个结论能被数值可靠地测出来的工具。

## 6. 给自己的三条红线

1. **不要再改 η 的结构。** 四因子已经够了。现在要做的是跑实验、看 η⁺ 在当前动作集上能不能把关键动作排序稳定。**排序稳定 = 设计成功**，不要折腾新因子。
2. **不要忘了主线是 Fig3 → Fig4 → Fig5-7 → Fig8/Fig9**。η 只服务于让这四张图变得锐利。任何不为这四张图服务的讨论，都暂缓。
3. **讲的时候按"目的 → 工具 → 细节"的顺序**。先说"我要找关键动作"，再说"所以我用 η⁺ 作为尺子"，最后才谈"η⁺ 的四个因子各是什么"。反过来讲别人会糊涂。
