# η⁺ 时延效率模型：物理意义与理论依据手册

> 本文档把 `computeEtaPlus.m` 与 `createDelayConfig.m` 中散落的物理依据注释整理为一份统一的"物理依据手册"，作为论文写作与组会汇报的叙事原料。
>
> 核心原则：**物理意义和理论依据放在第一位，数学公式只是物理图像的"翻译"。** 汇报顺序应为：项目场景 → 物理现象 → 理论依据 → 因子需求 → 数学形式 → 参数取值。每一步都是上一步的必然推论，让公式"水到渠成"，而不是"凭空设计"。

## 1. 项目场景：CPS 三层架构与时延的物理来源

本项目研究信息物理系统（CPS）在级联失效下的鲁棒性，系统分为三层：

- **power bus（PB，物理层）**：发电机、负荷、断路器等一次设备。
- **NCC（Non-control Center，区域汇聚层）**：本地 PMU/RTU 数据汇聚与指令分发节点。
- **CC（Control Center，中心决策层）**：执行广域监控与紧急控制决策。

信息流是**双向闭环**的，时延也分别在两条链路上累积：

### 1.1 正向链路（测量–上报）：power bus → NCC → CC

| 阶段 | 物理过程 | 时延来源 | 对应模型量 |
|------|---------|---------|-----------|
| PB → NCC | PMU/RTU 采样发电机出力、母线电压、频率偏差 | 采样周期 + ADC 处理死区（~50 ms） | τ_m0（测量死区） |
| NCC → CC | 区域数据经路由器/交换机向上转发 | 排队 + 传输 + 单跳丢包 | p_hop（单跳丢包率） |
| CC | 中心广域决策计算 | 控制环带宽对应的相位余量约束 | a_m（测量时延曲率） |

### 1.2 反向链路（指令–执行）：CC → NCC → power bus

| 阶段 | 物理过程 | 时延来源 | 对应模型量 |
|------|---------|---------|-----------|
| CC → NCC | 切机/减载指令向下分发 | 与上行共享同一通信网络的丢包规律 | p_hop（同上） |
| NCC → PB | 指令解码并触发本地执行机构 | 断路器动作死区、AGC 阀门响应（~50 ms） | τ_e0（执行死区） |
| PB | 不同容量机组对端到端总时延的耐受能力不同 | WAMS 对最大机组耐受上限 ~0.7–1.0 s | τ_crit_max（最大机组临界时延） |

> **叙事要点**：把这两条链路画清楚、讲清楚，听众脑中自然形成"时延在哪里产生 / 在哪里造成损失 / 在哪里决定生死"三类影响——三类影响对应三个因子，**Φ_sat / Φ_loss / Φ_crit 的"三"就不再是凭空选择，而是物理通道的自然分类。**

## 2. 三类物理影响 → 三个因子的必然性

时延对系统的影响只能归为三类**不可合并**的物理通道：

| # | 物理通道 | 物理现象 | 函数形式（理论依据） | 因子 |
|---|---------|---------|------------------|------|
| 1 | 连续通道 | 控制环相位裕度被纯延迟侵蚀，可用出力按指数衰减 | 一阶环对纯延迟响应近似 e^{−aτ} | **Φ_sat** |
| 2 | 离散通道 | 每跳独立丢包事件累积，端到端可达性下降 | 独立事件几何累积 (1−p)^n | **Φ_loss** |
| 3 | 临界通道 | 总时延超过阈值后稳定性骤然丧失（"悬崖"），且阈值与机组容量正相关 | logistic（陡峭度 β 控制悬崖宽度） | **Φ_crit** |

三者**函数形式不同**（指数 / 几何 / logistic）、**作用机制不同**（衰减 / 丢失 / 崩溃）、**作用位置不同**（源 / 路径 / 节点），不能互相替代也不能合并。

**为什么相乘？** 三类影响在物理上**相互独立**——控制环带宽侵蚀、网络丢包、临界稳定性损失分别由三套不相关的机制驱动；按概率/效率独立事件的乘性叠加规则，η⁺ = Φ_sat · Φ_loss · Φ_crit。

## 3. 三因子的"物理→理论→公式→参数"四段式

### 3.1 Φ_sat：饱和因子（连续通道，作用于 power bus 出力）

- **物理现象**：测量与执行通道的额外时延会破坏控制环相位裕度，使发电机实际可用出力按指数衰减。
- **理论依据**：一阶控制环传递函数对纯延迟的响应近似 e^{−aτ}；带宽 ω_c 对应"半衰时延" ln 2 / ω_c。本项目取 ω_c ≈ 1.4 rad/s（典型 AGC 内环），故 a_m ≈ 1.5、a_e ≈ 1.2。
- **数学形式**：因为"小延迟无害、超过死区才衰减"，加 max(0, τ − τ₀) 死区项；死区取 PMU 采样周期与执行机构动作时间的 50 ms 量级。
  ```
  Φ_sat = exp( −a_m · max(0, τ_m − τ_m0) − a_e · max(0, τ_e − τ_e0) )
  ```
- **参数取值**（见 `createDelayConfig.m:50–53`）：

  | 参数 | 默认值 | 物理含义 | 取值依据 |
  |------|-------|---------|---------|
  | a_m | 1.5 (1/s) | 测量时延曲率 | ln 2 / 0.5 ≈ 1.39，略上调以撬开 baseline↔heavy 的 R₁ 差距 |
  | a_e | 1.2 (1/s) | 执行时延曲率 | 保留"测量比执行更敏感"的工程经验 |
  | τ_m0 | 0.05 s | 测量死区 | PMU 典型采样周期 50 ms |
  | τ_e0 | 0.05 s | 执行死区 | 断路器/AGC 动作死区 50 ms |

### 3.2 Φ_loss：链路丢包因子（离散通道，作用于 NCC↔CC 通信路径）

- **物理现象**：每经一跳通信链路都有独立丢包概率，跳数越多端到端可达性越差。
- **理论依据**：独立丢包事件 → 可达概率几何累积 (1−p)^n；电力骨干通信网受扰丢包率经验区间 1%–10%，取中位 3%。
- **数学形式**：
  ```
  Φ_loss = (1 − p_hop) ^ n_hops_total
  n_hops_total = (numel(up_path) − 1) + (numel(down_path) − 1)
  ```
  恰好对应 power bus → NCC → CC → NCC → power bus 的完整闭环跳数。
- **参数取值**（见 `createDelayConfig.m:57`）：

  | 参数 | 默认值 | 物理含义 | 取值依据 |
  |------|-------|---------|---------|
  | p_hop | 0.03 | 单跳丢包率 | 电力骨干网受扰丢包率中位 |

### 3.3 Φ_crit：临界 logistic 因子（临界通道，作用于 power bus 异质机组）

- **物理现象**：广域控制对延迟存在"临界悬崖"——总时延超过某阈值后稳定性骤然丧失；不同容量机组的耐受阈值不同（大机组耐受高、小机组耐受低）。
- **理论依据**：
  - 稳定性悬崖用 logistic 函数刻画最自然，陡峭度 β 控制悬崖宽度。
  - 机组容量与耐受阈值成正比是 WAMS 文献的经验规律，故 τ_crit_i = τ_crit_max · r_i。
  - r_i 用**极值归一化（方案 A）** 保持单位一致：r_i = P_g(i) / max_j P_g(j)。case39 算例中分母为 1000 MW（bus 39），所有 r_i ∈ [0.25, 1.0]，r_min 永不触发。
- **数学形式**：
  ```
  τ_crit_i = τ_crit_max · r_i
  Φ_crit   = ( 1 + exp(−β) ) / ( 1 + exp( β · ((τ_m + τ_e) − τ_crit_i) / τ_crit_i ) )
  ```
  *归一化要点*：分子 (1 + exp(−β)) 即原始 logistic 在 τ=0 处的分母，因此 Φ_crit(τ=0) = 1 严格成立。物理含义为"相对存活率"——以理想信道 (τ=0) 为基线度量延迟带来的临界稳定性损失；理论依据来自可靠性工程中的相对/条件存活概率写法。这样三因子在 τ=0 时同时取 1，η⁺ 在 no_delay 场景严格等于 1，避免 ~0.25% 的非物理基线偏差让 light 在轻微攻击下偶发 R₁ 高于 no_delay。
- **参数取值**（见 `createDelayConfig.m:62–64`）：

  | 参数 | 默认值 | 物理含义 | 取值依据 |
  |------|-------|---------|---------|
  | τ_crit_max | 0.8 s | 最大机组临界总时延 | WAMS 文献 ~0.7–1.0 s 的中位 |
  | β | 6 | logistic 陡峭度 | 在 τ ≈ τ_crit 邻域内悬崖宽度约 ±15% |
  | r_min | 0.05 | 归一化下界 | 防止 r_i = 0 时 τ_crit_i → 0；case39 不触发，仅兼容其他算例 |

> **应用位置**：该因子作用在 power bus 端的异质机组，制造"小机组先崩、大机组后崩"的协同效应——这正是本项目要研究的级联机理。

## 4. 标准答辞：预设质疑与回答

### Q1：为什么是三个因子，不是五个、十个？

按"物理通道分类"原则，时延对系统的影响只有三类不可合并的物理通道（连续 / 离散 / 临界），每类对应唯一的函数形式（指数 / 几何 / logistic）。

- **加因子**：如排队抖动、量化误差等，都可以**归约**到上述三类之一（抖动并入 Φ_sat 的等效 τ_m，量化并入 Φ_loss 的等效 p_hop），不需要独立项。
- **减因子**：去掉 Φ_loss 就忽略了 cyber 网络拓扑；去掉 Φ_crit 就抹平了机组异质性；去掉 Φ_sat 就丢失了控制环带宽约束。任何一个都会**遗漏**一条物理通道。

故"恰好三个"是充分且必要的。

### Q2：为什么三个因子是相乘（而不是相加）？

三类影响在物理上相互独立——控制环带宽侵蚀、网络丢包、临界稳定性损失分别由三套不相关的机制驱动。按独立事件概率/效率的乘性叠加规则，必须相乘。相加会破坏 η ∈ [0, 1] 的物理含义（效率），并暗示三种损失可以"互相抵消"。

### Q3：参数为什么取这些数值？

每个参数都有**工程经验或文献依据**（见第 3 节各小节的"参数取值"表）。论文方法章节应附完整参数表，每行注明"物理含义 + 经验来源 + 量级估计"，把数值的合理性摆在台面上。

### Q4：为什么 r_i 用极值归一化（方案 A）而不是均值/求和/PMAX？

- **极值归一化**保证 r_i ∈ (0, 1] 且单位无关，物理上对应"最大机组耐受 τ_crit_max，其他机组按容量比例缩放"。
- 用均值或求和会让 r_i 的尺度依赖于电网规模，丧失跨算例可比性。
- 用 mpc.gen 的 PMAX 列会引入"额定容量 vs 实际出力"的歧义，与本模型"运行点耐受"的物理意义不符。

## 5. 汇报建议

1. **开场不写公式**：先放一张 power bus ↔ NCC ↔ CC 的双向链路示意图，标出每段时延来源。
2. **再讲三种物理影响**：用一句话各概括一种通道现象（衰减 / 丢失 / 崩溃），并给出对应的工程经验值或文献依据。
3. **然后才引入数学**：明确说"基于上述三种物理通道，提出 η⁺ = Φ_sat · Φ_loss · Φ_crit，相乘是因为三类影响相互独立、概率/效率乘性叠加"。
4. **参数表附物理解释列**：每个参数都注明物理含义、经验来源、量级估计。
5. **代码与论文用同一份注释**：直接复用本文档与 `createDelayConfig.m` 头注释，避免代码与论文"两套话语"。
6. **预设质疑清单**：把第 4 节的问答放在材料末尾，主动展示而不是被动招架。
