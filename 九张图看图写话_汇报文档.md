# 🔬 CPPS 时延-级联仿真：九张结果图的"为什么"

> **用途**：明天早上汇报用，一图一页，每张图回答三个问题：**画了什么？看到了什么？为什么会这样？**
>
> **整体模型一句话**：我们在 IEEE 39 节点电网 + BA 无标度通信网的耦合系统上，把通信时延"塞进"级联失效的循环里——时延让发电机出力打折(η)→潮流变了→新的过载→新的故障→通信更烂→时延更大→正反馈闭环。

---

## 🌐 整体模型回顾（30 秒版）

```
攻击信息层最高介数节点
       │
       ▼
┌─────────────────────────────────────────────┐
│  内循环（结构性传播，直到稳定）              │
│  A. 信息层孤岛检查 → 没有CC的岛全灭        │
│  B. 正向传播(信息→电力)，概率 p=0.3         │
│  C. 电力层孤岛检查 → 没有发电机的岛全灭     │
│  D. 反向传播(电力→信息)，概率 p=0.3         │
│  → 直到无新故障                              │
└─────────────────┬───────────────────────────┘
                  │ 稳定后
                  ▼
┌─────────────────────────────────────────────┐
│  外循环（过载 + 时延反馈）                   │
│  F. 信息层过载检查（介数>容量→失效）         │
│  G. ★时延注入★ 计算每台发电机的η           │
│     P_actual = P_ref × η                    │
│  H. DC潮流 + 电力支路过载检查                │
│  → 有新故障？回内循环。无？级联结束。        │
└─────────────────────────────────────────────┘
```

**关键公式**：

| 符号 | 含义 | 公式 |
|------|------|------|
| η | 控制效率因子 | η = (1 - k_m · τ_m) × (1 - k_e · τ_e)，钳位到 [0,1] |
| τ_m | 测量时延 | τ_PB→nonCC + T_up(最短路径) |
| τ_e | 执行时延 | τ_nonCC→PB + T_down(最短路径) |
| R₁ | 负荷保持率 | R₁ = (存活负荷 × φ) / 初始总负荷，φ = min(1, ΣP_actual/ΣP_ref) |
| R₃ | 发电偏差度 | R₃ = √(mean((P_actual - P_ref)²/P_ref²)) |
| α | 容量裕度因子 | 容量 = (1+α) × 初始负载 |

**五个时延场景**：

| 场景 | scale | 大白话 |
|------|-------|--------|
| no_delay | 0 | 理想通信，时延=0，η=1 |
| light | 0.5 | 通信质量好 |
| baseline | 1.0 | 标准时延 |
| medium | 1.5 | 通信有点差 |
| heavy | 2.0 | 通信很差 |

**保证公平性的关键设计**：预生成随机数表 `rand_forward`/`rand_backward`，相同 `(α, trial)` 下所有时延场景用相同随机数，R₁的差异只归因于时延。

## 📊 图1：R₁ 箱线图 (Box Plot)

### 画了什么？
对每个 (α, 场景) 组合，画出 500/1000 次蒙特卡洛试验的 R₁ 分布。X轴是 α（从0到1），每个 α 下有 5 个并排的箱子（5个时延场景）。

### 看到了什么？
1. **α=0 时全部崩溃**：所有场景的 R₁ 挤在 0.05-0.10，箱子又矮又窄——系统必死无疑
2. **α 增大后 no_delay（蓝色）明显高于其他场景**：从 α≈0.3 开始，蓝箱子和其他颜色拉开差距
3. **高 α 区域出现大量下方离群点**：即使 α=0.8，仍有些试验 R₁ 掉到很低
4. **时延场景的箱子比 no_delay 更宽**：分布更散，不确定性更大

### 为什么会这样？

| 现象 | 因果解释 |
|------|----------|
| α=0 全崩溃 | 容量 = 初始负载，**任何**潮流重分配都过载。时延来不及生效，纯结构性过载就把系统灭了 |
| α 增大后分离 | no_delay 的 η=1，发电机全出力；有时延的 η<1，出力打折→潮流失衡→额外过载→多死一批节点。这个差距就是"时延造成的额外伤害" |
| 时延场景箱子更宽 | 不同 BA 拓扑下同一台发电机到 CC 的路径长度不同 → η 不同 → 潮流不同 → 级联路径完全分叉。**时延把拓扑差异放大了**，本来微不足道的差异，在反馈闭环里被滚雪球放大成截然不同的结局 |
| 高 α 仍有低 R₁ 离群值 | 某些 BA 拓扑恰好让关键发电机完全失去 CC 可达性 (η=0)，等于直接关掉一台机组，即使容裕充足也扛不住 |

> **一句话总结**：时延不仅降低了系统平均表现，还放大了结果的不确定性——这对运营者来说更危险，因为你"预测不准"了。

## 📊 图2：R₁ vs α 折线图（Q25-Q75 截尾均值）

### 画了什么？
每个场景一条线，X轴是 α，Y轴是 R₁ 的截尾均值（只用箱线图中箱体范围 Q25-Q75 内的数据取平均，排除了极端离群值的干扰）。

### 看到了什么？
1. **no_delay 曲线**从 ~0.06（α=0）单调递增到 ~0.87（α=1.0），表现完美
2. **有时延的4条曲线**明显低于 no_delay，在 α=0.5-0.8 区间 gap 最大
3. **gap 呈"中间大、两头小"的形态**：α 低时大家都崩，α 极高时大家都还行，差距在中间最明显

### 为什么会这样？

| 现象 | 因果解释 |
|------|----------|
| 低 α 曲线重合 | **地板效应**：结构性过载在时延注入之前就把系统毁了，η 的差异没有发挥空间 |
| 中间 α gap 最大 | **杠杆区**：系统刚好有足够余量扛住 η=1 的情况（no_delay），但扛不住 η<1 带来的额外过载（有时延）。这恰好是时延最"致命"的区间——差一点就能活，但时延把你推下悬崖 |
| 高 α gap 缩小 | 容裕足够大，即使 η<1 导致潮流变化，也不至于触发新过载。但注意 R₁ 仍然低于 no_delay，因为 φ 因子（delay penalty）始终存在 |
| 使用截尾均值 | 直接用算术平均会被少量灾难性试验拉偏，截尾均值更能反映"典型"表现 |

> **一句话总结**：时延的危害在系统"将死未死"的临界区域最大——恰好是运营者最依赖韧性评估的区间。

## 📊 图3：R₃ vs α 折线图

### 画了什么？
X轴是 α，Y轴是 R₃（发电机实际出力与参考出力的均方根偏差）。5 个场景 5 条线。

### 看到了什么？
1. **5条线严格分层，没有交叉**：heavy 最高（偏差最大），no_delay 最低（偏差最小）
2. **所有线随 α 递减**：α 越大，R₃ 越小
3. **no_delay 的 R₃ 在高 α 时趋近 0**：发电机几乎完美执行调度指令
4. **即使高 α，heavy 的 R₃ 仍然很高**

### 为什么会这样？

| 现象 | 因果解释 |
|------|----------|
| 严格分层不交叉 | R₃ 直接由 η 决定：η = (1-k_m·τ_m)(1-k_e·τ_e)。时延越大 → η 越小 → (P_actual-P_ref)/P_ref 偏差越大 → R₃ 越大。这是**确定性关系**，不像 R₁ 受潮流重分配的随机性影响 |
| 随 α 递减 | α 越大 → 过载越少 → 存活发电机越多 → 更多发电机保持可达(η>0) → 平均偏差自然更小 |
| no_delay R₃≈0 | 所有可达发电机 η=1 → P_actual=P_ref → 偏差=0。只有失去CC可达性的发电机(η=0)才贡献偏差 |
| heavy 高α仍高R₃ | 即使存活发电机多，每台的 η 仍然被高时延压低。**R₃揭示了R₁看不到的隐藏退化**：负荷存活率看起来还行，但发电机都在"带病运行" |

> **一句话总结**：R₃ 是"X光片"——R₁ 说"病人还活着"，R₃ 说"但内脏功能已经严重受损"。时延越重，"隐性伤害"越大。

## 📊 图4：延迟惩罚热力图 (Delay Penalty Heatmap)

### 画了什么？
热力图，X轴是级联轮次 (Cascade Round)，Y轴是 α。颜色 = ΔR₁ = R₁^{no_delay} - R₁^{heavy}，越亮说明 no_delay 比 heavy 多存活的负荷越多（时延造成的"额外伤害"越大）。

### 看到了什么？
1. **第1轮全黑** (ΔR₁≈0)：无论什么 α，第1轮时延都没有影响
2. **第2轮在中高α区域突然变亮**：惩罚陡然出现
3. **第3-5轮持续加深变亮**：惩罚不断累积
4. **第5轮之后颜色稳定**：基本"定格"了
5. **低α区域始终是黑色**：时延无所谓

### 为什么会这样？

| 现象 | 因果解释 |
|------|----------|
| 第1轮全黑 | **预生成随机数表**保证了所有场景在第1轮的结构性传播（正向/反向概率传播、岛屿检查）完全相同。时延注入发生在外循环的步骤G，第1轮还没执行到这一步 |
| 第2轮突然变亮 | **反馈闭环第一次点火**：第1轮结束后外循环首次注入时延，heavy 场景的 η<1 使发电机降出力 → 潮流改变 → 触发 no_delay 不会触发的过载 → 两条级联路径从此分叉。这就是"阈值跨越"——不是渐变，是突变 |
| 第3-5轮持续加深 | **正反馈放大**：第2轮的额外故障 → 通信网进一步破坏 → 路径更长/更多不可达 → τ 更大 → η 更低 → 更多过载 → 更多故障... 雪球越滚越大 |
| 第5轮后饱和 | 反馈闭环已经**耗尽了脆弱点**：该断的线都断了，该死的节点都死了，剩下的都足够强壮。或者系统已经全面崩溃，没什么可以再坏的了 |
| 低α始终黑色 | **地板效应**：两个场景都全面崩溃了，R₁ 差不多都是 0，差值自然≈0 |
| 中高α最亮 | **杠杆区**：这里时延的"一推"就能决定系统生死。no_delay 刚好活下来，heavy 刚好被推下悬崖 |

> **一句话总结**：时延伤害不是"一直在发生"，而是在**第2轮突然引爆**，然后像火灾一样在3-5轮内蔓延到不可收拾。第1轮是唯一的安全窗口。

## 📊 图5：敏感性分析 R₁ vs α（全场景对比）

### 画了什么？
在图2的基础上，增加了5个工程动作(A1-A5)的曲线。共7条线：no_delay（上界）、heavy（下界）、A1-A5（改善措施）。

**5个工程动作是什么？**

| 动作 | 干了什么 | 大白话 |
|------|----------|--------|
| A1: 带宽升级 | 链路速率 10→100 Mbps | 换更快的网线 |
| A2: 端点处理优化 | CC和nonCC的tx/rx时延减半 | 升级路由器芯片 |
| A3: 转发优化 | 转发时延减半 | 优化中间节点性能 |
| A4: 测量接口加速 | 测量时延 200→20 ms | 升级传感器接口 |
| A5: 执行接口加速 | 执行时延 240→30 ms | 升级执行器接口 |

### 看到了什么？
1. 所有动作曲线都在 heavy 和 no_delay 之间
2. 某些动作（例如A4或A5）明显更接近 no_delay
3. 某些动作（例如A1或A3）改善很小，几乎贴着 heavy
4. 在中间α区域（0.4-0.7），动作之间的差距最明显

### 为什么会这样？

关键：看时延公式的构成——

```
τ_m = τ_PB→nonCC(测量接口时延, heavy=200ms) + T_up(路径时延, 约几ms)
τ_e = τ_nonCC→PB(执行接口时延, heavy=240ms) + T_down(路径时延, 约几ms)
```

| 现象 | 因果解释 |
|------|----------|
| A4/A5 效果最好 | 测量接口时延(200ms)和执行接口时延(240ms)是端到端时延的**绝对主导项**，占比超过95%！降低它们直接大幅降低 τ_m/τ_e → η 大幅提升 |
| A1 效果很小 | 链路传输时延 T_link = S/R + d/V ≈ 几μs 到几ms，相比场侧接口的 200ms 简直微不足道。带宽翻10倍也只省了几μs |
| A2/A3 效果中等 | 节点处理时延（几ms到十几ms）在多跳路径上能累积一点，但总量远小于场侧接口时延 |
| 中间α差距最大 | 同样的"杠杆区"效应：只有在这个区间，η 的小幅提升才能改变过载/不过载的判定 |

> **一句话总结**：别花大价钱升级骨干网带宽了，**升级传感器接口和执行器接口**才是性价比最高的。瓶颈在"最后一公里"。

## 📊 图6：恢复比例热力图 (Recovery % Heatmap)

### 画了什么？
热力图，X轴是 α，Y轴是5个动作(A1-A5)。颜色 = 恢复比例 = (R₁_动作 - R₁_heavy) / (R₁_nodelay - R₁_heavy) × 100%。

**含义**：如果 = 100%，说明这个动作完全消除了时延的伤害（恢复到 no_delay 水平）；= 0% 说明毫无改善。

### 看到了什么？
1. A4/A5 那两行整体偏亮（恢复比例高）
2. A1/A3 那两行整体偏暗（恢复比例低）
3. 低 α 区域所有动作都不太亮（或数值不稳定）
4. 中高 α 区域差异最明显

### 为什么会这样？

| 现象 | 因果解释 |
|------|----------|
| A4/A5 行最亮 | 同图5的解释：它们削减的是时延的主导分量（场侧接口），η 提升幅度最大 |
| A1/A3 行最暗 | 它们削减的是时延的微小分量（链路传输/转发），杯水车薪 |
| 低α不亮 | 分母 (R₁_nodelay - R₁_heavy) ≈ 0，两个场景都崩溃了，改善空间本身就不存在 |
| 某些格子超过100%或为负 | 数值不稳定区域（分母极小），不必过度解读 |

> **一句话总结**：这张图是"投资回报率地图"——告诉你在哪个 α 区间、用哪个动作，获得最大的时延伤害恢复。

## 📊 图7：动作排名柱状图 (Action Ranking)

### 画了什么？
5 根柱子，按平均恢复比例（α≥0.3 范围内的均值）从高到低排列。柱子上标数值。

### 看到了什么？
- 有明确的排名顺序，某几个动作明显领先

### 为什么排名是这个顺序？

回到时延公式做"成分分析"：

```
η = (1 - k_m · τ_m) × (1 - k_e · τ_e)

其中：
τ_m = 200ms(测量接口) + ~2ms(路径) ≈ 202ms      → k_m × τ_m = 0.80 × 0.202 = 0.162
τ_e = 240ms(执行接口) + ~1ms(路径) ≈ 241ms      → k_e × τ_e = 0.60 × 0.241 = 0.145

η ≈ (1 - 0.162) × (1 - 0.145) = 0.838 × 0.855 = 0.717 (heavy场景)
```

**A4 把 τ_m 降到 20ms + ~2ms → k_m·τ_m = 0.018 → η 提升到 ~0.84**
**A5 把 τ_e 降到 30ms + ~1ms → k_e·τ_e = 0.019 → η 提升到 ~0.82**
**A1 把链路速率翻10倍 → 路径从 ~2ms 降到 ~0.2ms → 几乎没变**

所以排名的本质就是：**谁能让 η 提升最多，谁就排名最高。** 而 η 的提升幅度取决于该动作削减的时延分量在总时延中的占比。

> **一句话总结**：排名揭示了时延的"瓶颈在哪里"——绝大部分时延卡在物理侧接口，不在通信网内部。

## 📊 图8：对比实验柱状图（2×2 设计）

### 画了什么？
4 根柱子，代表 4 种条件组合：

| 条件 | 时间（Cascade Round） | 动作 | 含义 |
|------|----------------------|------|------|
| C1 | 最佳时间（delay penalty最大的轮次，如Round 2-3） | 最佳动作（排名第1） | 天时地利 |
| C2 | 最佳时间 | 最差动作（排名最后） | 只有天时 |
| C3 | 最差时间（delay penalty最小的轮次，如Round 7+） | 最佳动作 | 只有地利 |
| C4 | 最差时间 | 最差动作 | 啥也没有 |

**注意**：这里的"时间"是 Cascade Round（级联轮次），不是 α！最佳/最差时间是从图4热力图中取 delay penalty 最大/最小的轮次。

### 看到了什么？
- C1 的恢复比例最高
- C4 的恢复比例最低
- C2 和 C3 在中间

### 为什么会这样？

| 条件 | 因果解释 |
|------|----------|
| C1 最高 | 在反馈闭环杠杆最大的时刻（Round 2-3，正反馈刚点火）采取效果最强的动作 → η 的改善直接阻止了"雪崩"的启动，一步到位 |
| C4 最低 | 在反馈闭环已经结束的时刻（Round 7+）用最弱的动作 → 系统已经"定型"了，做什么都晚了，而且动作本身也不够力 |
| C2 vs C3 | 谁更高取决于时间和动作哪个因素更重要。如果 C1-C2 > C3-C4 的多，说明"选对动作"的边际贡献更大；反之说明"选对时间"更重要 |
| 如果 C1 > C2+C3-C4 | 存在**协同效应** (synergy)：好时机+好动作不是简单叠加，而是产生了超额收益。因为在关键时刻的一小步改善，通过正反馈被放大成了巨大的结果差异 |

> **一句话总结**：这是本文最重要的实践指导——**在级联第2-3轮时果断升级传感器/执行器接口，效果远远好于"等级联快结束了再做什么"。**

## 📊 图9：对比实验详细热力图

### 画了什么？
两个并排热力图，X轴是 Cascade Round，Y轴是 α：
- **左图**：最佳动作的恢复比例 Recovery%，在每个 (round, α) 点上的值
- **右图**：最差动作的恢复比例 Recovery%
- 绿色虚线标注了"最佳干预时间"所在的轮次

### 看到了什么？

2. **两图都在 Round 2-3 × 中高α 区域最亮**：这是"黄金干预窗口"
3. **Round 1 列两图都暗**：第1轮没有时延惩罚可消除
4. **晚期轮次颜色变淡/不稳定**：系统已经定型

### 为什么会这样？

| 现象 | 因果解释 |
|------|----------|
| 最佳动作全面更亮 | 它削减了时延的主导分量，在**任何**时间和 α 组合下都能更有效地提升 η |
| Round 2-3 × 中高α 最亮 | Round 2-3 是正反馈刚点火的时刻——干预的"杠杆"最大。中高α 是系统在生死边缘——η 的小改善能决定"过载还是不过载" |
| Round 1 暗 | 所有场景在第1轮完全相同（预生成随机数保证），没有时延造成的差异可以被恢复 |
| 晚期轮次淡化 | 正反馈闭环已经饱和或系统已崩溃，干预窗口已关闭。这就像山火烧了80%之后再去灭火——能救的很有限 |

> **一句话总结**：图9是"精准干预指南"——它把图8的宏观结论展开到每一个 (轮次, α) 坐标上，告诉你在哪里干预最值、在哪里干预已经太晚。

## 🎯 三句话总结（给老师/答辩用）

1. **时延不是"事后减分项"，而是级联失效的"正反馈加速器"**——它在第2轮引爆，在3-5轮内通过闭环放大造成不可逆伤害。

2. **时延的瓶颈在"最后一公里"**——传感器测量接口和执行器指令接口占端到端时延的95%以上，升级骨干网带宽几乎无效。

3. **干预的时机和手段必须同时正确**——在级联早期(Round 2-3)对场侧接口进行加速，能获得最大的系统韧性恢复；错过窗口或选错手段，效果大打折扣。
