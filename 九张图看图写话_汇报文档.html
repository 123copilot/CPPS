🌐 整体模型回顾（30秒版）

一句话：在 IEEE 39节点电网 + BA无标度通信网的耦合系统上，把通信时延"塞进"级联失效的循环里——时延让发电机出力打折(η) → 潮流变了 → 新的过载 → 新的故障 → 通信更烂 → 时延更大 → 正反馈闭环。

攻击信息层最高介数节点 │ ▼ ┌────────────────────────────────────────────┐ │ 内循环（结构性传播，直到稳定） │ │ A. 信息层孤岛检查 → 没有CC的岛全灭 │ │ B. 正向传播(信息→电力)，概率 p=0.3 │ │ C. 电力层孤岛检查 → 没有发电机的岛全灭 │ │ D. 反向传播(电力→信息)，概率 p=0.3 │ │ → 直到无新故障 │ └───────────────────┬────────────────────────┘ │ 稳定后 ▼ ┌────────────────────────────────────────────┐ │ 外循环（过载 + 时延反馈） │ │ F. 信息层过载检查（介数>容量→失效） │ │ G. ★时延注入★ 计算每台发电机的η │ │ P_actual = P_ref × η │ │ H. DC潮流 + 电力支路过载检查 │ │ → 有新故障？回内循环。无？级联结束。 │ └────────────────────────────────────────────┘
符号含义公式/定义
η控制效率因子η = (1 − k_m · τ_m) × (1 − k_e · τ_e)，钳位到 [0,1]
τ_m测量时延τ_PB→nonCC + T_up(最短路径)
τ_e执行时延τ_nonCC→PB + T_down(最短路径)
R₁负荷保持率(存活负荷 × φ) / 初始总负荷, φ = min(1, ΣP_actual/ΣP_ref)
R₃发电偏差度√(mean((P_actual − P_ref)² / P_ref²))
α容量裕度容量 = (1+α) × 初始负载
时延场景Scale大白话
no_delay0理想通信，时延=0，η=1
light0.5通信质量好
baseline1.0标准时延
medium1.5通信有点差
heavy2.0通信很差
公平性保证：预生成随机数表 rand_forward / rand_backward，相同 (α, trial) 下所有时延场景用相同随机数。R₁ 的差异只归因于时延配置不同。
1 R₁ 箱线图 (Box Plot) — 时延如何影响结果的分布？

画了什么？

对每个 (α, 场景) 组合，画出 Monte Carlo 试验的 R₁ 分布。X轴是 α（0→1），每个 α 下有 5 个并排的箱子（5个时延场景）。

看到了什么？

① α=0 时全部崩溃，箱子又矮又窄
② α 增大后 no_delay（蓝色）明显高于其他场景
③ 高 α 区域出现大量下方离群点
④ 时延场景的箱子比 no_delay 更宽（不确定性更大）

为什么会这样？

现象因果解释
α=0 全崩溃容量=初始负载，任何潮流重分配都过载。时延来不及生效，纯结构性过载就把系统灭了
α 增大后分离no_delay 的 η=1，发电机全出力；有时延的 η<1，出力打折→潮流失衡→额外过载→多死一批节点
时延场景箱子更宽不同 BA 拓扑下同一台发电机到 CC 的路径长度不同→η不同→潮流不同→级联路径完全分叉。时延把拓扑差异放大了
高α仍有低R₁离群值某些 BA 拓扑恰好让关键发电机完全失去 CC 可达性(η=0)，等于关掉一台机组
💡 时延不仅降低了系统平均表现，还放大了结果的不确定性——你"预测不准"了，这对运营者更危险。
2 R₁ vs α 折线图 — 时延伤害的"趋势"

画了什么？

每个场景一条线，X轴 α，Y轴 R₁ 的 Q25-Q75 截尾均值（排除离群值干扰）。

看到了什么？

① no_delay 单调递增到约 0.87
② 有时延的曲线明显低于 no_delay
③ gap 呈"中间大、两头小"的形态

为什么会这样？

现象因果解释
低α曲线重合地板效应：结构性过载在时延注入之前就摧毁了系统，η 的差异没有发挥空间
中间α gap最大杠杆区：系统刚好有余量扛住 η=1 但扛不住 η<1 的额外过载。时延把你推下悬崖
高α gap缩小容裕足够大，η<1 导致的潮流变化不至于触发新过载。但 R₁ 仍低于 no_delay（φ 因子始终存在）
💡 时延的危害在系统"将死未死"的临界区域最大——恰好是运营者最依赖韧性评估的区间。
3 R₃ vs α 折线图 — 隐性伤害的"X光片"

画了什么？

X轴 α，Y轴 R₃（发电机实际出力与参考出力的 RMSD 偏差）。5个场景5条线。

看到了什么？

① 5条线严格分层，没有交叉：heavy最高，no_delay最低
② 所有线随 α 递减
③ no_delay 在高α时 R₃≈0
④ heavy 在高α时 R₃ 仍然很高

为什么会这样？

现象因果解释
严格分层不交叉R₃ 直接由 η 决定，而 η 由时延确定性决定，不像 R₁ 受潮流重分配的随机性影响
随α递减α越大→过载越少→存活发电机越多→更多保持可达→平均偏差更小
no_delay R₃≈0所有可达发电机 η=1 → P_actual=P_ref → 偏差=0
heavy高α仍高R₃即使存活多，每台的 η 仍被高时延压低。负荷存活率看起来还行，但发电机都在"带病运行"
💡 R₁ 说"病人还活着"，R₃ 说"但内脏功能已经严重受损"。时延越重，隐性伤害越大。
4 延迟惩罚热力图 — 时延伤害的"时空地图"

画了什么？

X轴级联轮次(Round)，Y轴α。颜色 = ΔR₁ = R₁no_delay − R₁heavy，越亮=时延伤害越大。

看到了什么？

① 第1轮全黑（ΔR₁≈0）
② 第2轮突然变亮（中高α区域）
③ 第3-5轮持续加深
④ 第5轮后颜色稳定
⑤ 低α始终黑色

为什么会这样？

现象因果解释
第1轮全黑预生成随机数表保证所有场景在第1轮的结构性传播完全相同。时延注入在外循环步骤G，第1轮还没执行到
第2轮突然亮反馈闭环第一次点火！外循环首次注入时延，η<1→潮流改变→触发no_delay不会有的过载→两条路径分叉。是突变不是渐变
第3-5轮加深正反馈放大：额外故障→通信更坏→τ更大→η更低→更多过载→雪球越滚越大
第5轮后饱和反馈闭环已耗尽脆弱点：该断的都断了，剩下的足够强壮（或系统已全面崩溃）
低α始终黑两个场景都全面崩溃，R₁差≈0，地板效应
💡 时延伤害在第2轮突然引爆，然后像火灾一样在3-5轮内蔓延到不可收拾。第1轮是唯一安全窗口。
5 敏感性分析 R₁ vs α — 五个"药方"哪个最管用？

画了什么？

7条线：no_delay(上界)、heavy(下界)、A1-A5(5个工程动作)。

动作干了什么大白话
A1: 带宽升级10→100 Mbps换更快的网线
A2: 端点处理优化tx/rx时延减半升级路由器芯片
A3: 转发优化转发时延减半优化中间节点
A4: 测量接口加速200→20 ms升级传感器接口
A5: 执行接口加速240→30 ms升级执行器接口

为什么 A4/A5 效果最好，A1 效果最差？

τ_m = 200ms(测量接口,主导!) + ~2ms(路径) ≈ 202ms τ_e = 240ms(执行接口,主导!) + ~1ms(路径) ≈ 241ms → 场侧接口占时延的 95%以上 → A4/A5 削减主导项 → 大幅降τ → η大幅上升 → A1 把链路速率翻10倍 → 路径从2ms降到0.2ms → 几乎没变
💡 别花大价钱升级骨干网带宽了，升级传感器接口和执行器接口才是性价比最高的。瓶颈在"最后一公里"。

7 动作排名柱状图 — 时延瓶颈在哪？

画了什么？

5根柱子，按平均恢复比例（α≥0.3 均值）从高到低排列。

为什么排名是这个顺序？

η = (1 − k_m · τ_m) × (1 − k_e · τ_e) heavy场景下: k_m × τ_m = 0.80 × 0.202 = 0.162 ← 测量接口占大头 k_e × τ_e = 0.60 × 0.241 = 0.145 ← 执行接口占大头 A4 把 τ_m 降到 22ms → k_m·τ_m = 0.018 → η 大幅提升 A5 把 τ_e 降到 31ms → k_e·τ_e = 0.019 → η 大幅提升 A1 把路径从 2ms→0.2ms → → η 基本不变
💡 排名的本质：谁能让 η 提升最多，谁排名最高。时延瓶颈在物理侧接口，不在通信网内部。

🎯 三句话总结（给老师 / 答辩用）

  1. 时延不是"事后减分项"，而是级联失效的"正反馈加速器"——它在第2轮引爆，在3-5轮内通过闭环放大造成不可逆伤害。
  2. 时延的瓶颈在"最后一公里"——传感器测量接口和执行器指令接口占端到端时延的95%以上，升级骨干网带宽几乎无效。
  3. 干预的时机和手段必须同时正确——在级联早期(Round 2-3)对场侧接口进行加速，能获得最大的系统韧性恢复；错过窗口或选错手段，效果大打折扣。
